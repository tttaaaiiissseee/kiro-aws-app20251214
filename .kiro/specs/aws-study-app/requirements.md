# 要件定義書

## 概要

AWS学習アプリは、AWSサービスの知識を体系的に整理・管理し、効率的な学習を支援するWebアプリケーションです。ユーザーはAWSサービスに関するメモ、リンク、画像を保存し、サービス間の関連性を視覚的に把握できます。カテゴリ別の整理機能により、膨大なAWSサービスを構造化して学習できます。

## 用語集

- **AWS学習アプリ**: AWSサービス学習支援Webアプリケーション
- **サービスエントリ**: 個別のAWSサービスに関する情報エントリ
- **カテゴリ**: AWSサービスの分類（Compute、Storage、Network、Security、ML等）
- **メモ**: テキスト、リンク、画像を含むユーザーの学習メモ
- **関連マップ**: AWSサービス間の関連性を示すビジュアル表現
- **比較テーブル**: 複数のAWSサービスを比較するための表形式表示

## 要件

### 要件 1

**ユーザーストーリー:** AWS学習者として、AWSサービスのサービスエントリを作成・管理したい。各サービスに関する知識を体系的に整理するため。

#### 受入基準

1. ユーザーが新しいサービスエントリを作成するとき、AWS学習アプリはサービス名、カテゴリ、説明を保存する
2. ユーザーが既存のサービスエントリを編集するとき、AWS学習アプリは情報を更新し、変更タイムスタンプを保持する
3. ユーザーがサービスエントリを削除するとき、AWS学習アプリは関連するすべてのメモと関係性を削除する
4. ユーザーがサービスエントリを検索するとき、AWS学習アプリはサービス名または説明に一致する結果を返す
5. AWS学習アプリはサービス名がシステム内で一意であることを検証する

### 要件 2

**ユーザーストーリー:** AWS学習者として、サービスエントリに様々な種類のメモを追加したい。テキストノート、有用なリンク、視覚的な参考資料を含む包括的な情報を記録するため。

#### 受入基準

1. ユーザーがサービスエントリにテキストメモを追加するとき、AWS学習アプリはタイムスタンプ付きでテキスト内容を保存する
2. ユーザーがリンクメモを追加するとき、AWS学習アプリはURL形式を検証し、オプションの説明と共にリンクを保存する
3. ユーザーが画像メモをアップロードするとき、AWS学習アプリは画像ファイルを保存し、サムネイルを生成する
4. ユーザーがメモを表示するとき、AWS学習アプリはタイプ表示と共に時系列順で表示する
5. ユーザーがメモを削除するとき、AWS学習アプリは内容と関連ファイルを削除する

### 要件 3

**ユーザーストーリー:** AWS学習者として、AWSサービスをカテゴリ分けしたい。機能領域別にサービスを整理・閲覧するため。

#### 受入基準

1. ユーザーがサービスエントリにカテゴリを割り当てるとき、AWS学習アプリは指定されたカテゴリとサービスを関連付ける
2. ユーザーがカテゴリ別にサービスを表示するとき、AWS学習アプリはそれぞれのカテゴリの下にグループ化されたすべてのサービスを表示する
3. ユーザーが新しいカテゴリを作成するとき、AWS学習アプリはカテゴリ名が一意であることを検証する
4. AWS学習アプリはCompute、Storage、Network、Security、MLを含む事前定義カテゴリを提供する
5. ユーザーがカテゴリでサービスをフィルタするとき、AWS学習アプリは選択されたカテゴリに属するサービスのみを表示する

### 要件 4

**ユーザーストーリー:** AWS学習者として、AWSサービス間の関係性を視覚化したい。サービスがどのように統合され、連携するかを理解するため。

#### 受入基準

1. ユーザーが2つのサービス間の関係を作成するとき、AWS学習アプリは関係タイプと共に双方向の接続を保存する
2. ユーザーが関連マップを表示するとき、AWS学習アプリはラベル付きエッジで接続されたノードとしてサービスを表示する
3. ユーザーが関連マップ内のサービスノードをクリックするとき、AWS学習アプリは接続されたサービスをハイライトし、関係の詳細を表示する
4. ユーザーが関係を削除するとき、AWS学習アプリは関連マップを即座に更新する
5. AWS学習アプリは「統合する」「依存する」「代替となる」を含む関係タイプをサポートする

### 要件 5

**ユーザーストーリー:** AWS学習者として、複数のAWSサービスを並べて比較したい。違いを理解し、特定のユースケースに適切なサービスを選択するため。

#### 受入基準

1. ユーザーが比較のために複数のサービスを選択するとき、AWS学習アプリは比較テーブル形式で表示する
2. 比較テーブルを表示するとき、AWS学習アプリは料金モデル、ユースケース、制限を含む主要属性を表示する
3. ユーザーが比較基準を追加するとき、AWS学習アプリは比較のためのカスタム属性定義を許可する
4. ユーザーが比較をエクスポートするとき、AWS学習アプリはPDFまたはCSV形式でダウンロード可能なレポートを生成する
5. AWS学習アプリは読みやすさのため、同時比較を最大5サービスに制限する

### 要件 6

**ユーザーストーリー:** AWS学習者として、知識ベースを検索・フィルタしたい。学習セッション中に関連情報を素早く見つけるため。

#### 受入基準

1. ユーザーが検索語を入力するとき、AWS学習アプリはサービス名、説明、メモ内容を横断して検索する
2. ユーザーがカテゴリフィルタを適用するとき、AWS学習アプリは選択されたカテゴリからの結果のみを表示する
3. ユーザーが検索結果をソートするとき、AWS学習アプリは関連度、アルファベット順、または最終更新日で並び替える
4. 検索で結果が返されないとき、AWS学習アプリは代替検索語を提案するか、人気のサービスを表示する
5. AWS学習アプリは簡単な識別のため、結果内の検索語をハイライトする

### 要件 7

**ユーザーストーリー:** AWS学習者として、アプリケーションをDockerコンテナで実行したい。異なる環境でアプリケーションを簡単にデプロイ・管理するため。

#### 受入基準

1. ユーザーがDockerイメージをビルドするとき、AWS学習アプリは必要なすべての依存関係と設定を含む
2. ユーザーがDockerコンテナを実行するとき、AWS学習アプリはWebサーバーとデータベースサービスを開始する
3. コンテナが開始するとき、AWS学習アプリは必要に応じてデータベーススキーマとシードデータを初期化する
4. ユーザーがコンテナを停止するとき、AWS学習アプリはサービスを適切にシャットダウンし、データを保持する
5. AWS学習アプリはDockerボリュームを通じて永続的なデータストレージをサポートする

### 要件 8

**ユーザーストーリー:** AWS学習者として、レスポンシブなWebインターフェースが欲しい。デスクトップやモバイルを含む異なるデバイスで学習資料にアクセスするため。

#### 受入基準

1. ユーザーがデスクトップでアプリケーションにアクセスするとき、AWS学習アプリはサイドバーナビゲーション付きの完全なインターフェースを表示する
2. ユーザーがモバイルでアプリケーションにアクセスするとき、AWS学習アプリは折りたたみ可能なナビゲーションでレイアウトを適応させる
3. ユーザーがモバイルでタッチジェスチャーを操作するとき、AWS学習アプリはスワイプとタップに適切に応答する
4. 画面サイズが変更されるとき、AWS学習アプリは機能を失うことなくレイアウトを調整する
5. AWS学習アプリはすべてのデバイスサイズで一貫した視覚的デザインを維持する
